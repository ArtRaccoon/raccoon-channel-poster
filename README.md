# Raccoon Channel Link Editor Bot

Telegram-бот, который автоматически редактирует новые опубликованные посты в каналах и добавляет в конец поста блок полезных ссылок в виде цитаты.

## Что делает бот

- Получает `channel_post` update из Telegram-канала.
- Находит канал в SQLite-базе.
- Проверяет, включена ли автообработка и задан ли блок ссылок.
- Редактирует текст поста через `editMessageText` или caption медиа через `editMessageCaption`.
- Добавляет ссылки HTML-блоком `<blockquote>`.
- Не публикует посты сам, не делает repost и не создаёт новые сообщения в канале.
- Только редактирует `text` или `caption` существующего поста.

## Что больше не поддерживается

Этот проект больше не является scheduler/posting bot. Старые сценарии создания постов, расписаний, черновиков и batch-загрузок отключены из основного UI и из подключения роутеров.

## Требования

- Python 3.11+
- SQLite
- aiogram 3.x
- Telegram bot token
- Бот должен быть администратором канала.
- Критичное право: **редактирование сообщений** (`can_edit_messages`).
- Право публикации (`can_post_messages`) желательно, но для добавления ссылок в уже опубликованные пользователем посты не является основным.

## Настройка

Создайте `.env` или задайте переменные окружения:

```env
BOT_TOKEN=123456:telegram-token
OWNER_IDS=123456789
DATABASE_PATH=data/bot.db
LOG_LEVEL=INFO
# PROXY_URL можно использовать как раньше
PROXY_URL=
```

`PROXY_URL`, `AiohttpSession` и proxy-логика сохранены без изменения.

Важно: для работы нужны `channel_post` updates, поэтому polling/webhook не должен ограничивать `allowed_updates` списком без `channel_post`.

## Запуск

```bash
python -m bot.main
```

Для systemd можно продолжать использовать существующий запуск приложения: важно, чтобы рабочая директория указывала на корень проекта, а переменные окружения были доступны процессу.

## Как пользоваться

1. Добавьте бота в Telegram-канал.
2. Выдайте права администратора и право редактировать сообщения.
3. Откройте бота в личке.
4. Перейдите в **⚙️ Настройки → 📣 Каналы → ➕ Добавить канал**.
5. Отправьте username канала, например `@my_channel`.
6. Перейдите в **⚙️ Настройки → 🔗 Полезные ссылки** и задайте блок ссылок.
7. Опубликуйте пост в канал вручную.
8. Бот автоматически отредактирует пост и добавит блок ссылок.

Пример блока ссылок:

```text
✦ Донат | https://example.com/donate
✦ Boosty | https://boosty.to/example
✦ Pinterest | https://pinterest.com/example
✦ Telegram | https://t.me/example
```

Итоговый блок в посте:

```html
<blockquote>
✦ Полезные ссылки
✦ Донат | https://example.com/donate
✦ Boosty | https://boosty.to/example
</blockquote>
```

## Меню

Обычный пользователь видит:

- ⚙️ Настройки

Владелец из `OWNER_IDS` видит:

- ⚙️ Настройки
- 🛠 Админ-панель

В настройках доступны:

- 📣 Каналы
- 🔗 Полезные ссылки
- ✅ Автообработка: Вкл/Выкл
- 🧪 Тест редактирования
- ⬅️ Назад

## Лимиты Telegram

Бот не редактирует сообщение, если после добавления блока превышается лимит Telegram:

- text: 4096 символов;
- caption: 1024 символа.

В этом случае событие пишется в `edit_logs`, а владельцу канала по возможности отправляется уведомление в личку.

Бот редактирует сообщение заново через `HTML` parse_mode. Исходное форматирование поста может быть упрощено, если оно было сделано через Telegram entities.

## Логи

Технические логи пишутся в `logs/bot.log`.

События редактирования пишутся в таблицу `edit_logs` со статусами вроде:

- `success`
- `skipped_no_links`
- `skipped_already_has_block`
- `skipped_no_text_or_caption`
- `error_limit`
- `error_telegram`

OWNER-пользователям доступна кнопка **🧾 Логи редактирования**.
