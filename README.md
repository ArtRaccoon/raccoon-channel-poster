# raccoon-channel-poster

Telegram-бот для публикации постов в каналы пользователей (текст и фото) с черновиками, расписанием, предпросмотром и безопасными инструментами эксплуатации.

## Возможности

- Добавление канала и проверка прав бота/пользователя.
- Создание черновика из текста, фото или фото+подпись.
- Реальный предпросмотр будущего поста в личке перед публикацией: текст/фото, URL-кнопки и подпись канала отображаются так же, как в канале.
- URL-кнопки для постов: до 8 кнопок в формате `Текст | https://example.com`.
- Автоподпись для каждого канала: подпись добавляется к тексту или caption при публикации и предпросмотре.
- Замена фото в черновике: photo-пост обновляет фото, text-only пост можно превратить в photo-пост, если текст помещается в caption.
- Просмотр `📝 Черновики` и `🕘 Последние посты`.
- Поиск `🔎 Поиск постов` по текстам своих постов с быстрым открытием найденных записей.
- Дублирование постов в новый черновик.
- Перепубликация опубликованного поста в другой канал через новый черновик и подтверждение.
- Очистка старых черновиков старше 30 дней только для текущего пользователя.
- Inline-подтверждение: публикация, редактирование текста, URL-кнопки, замена фото, выбор канала, сохранение, планирование, отмена.
- Планирование публикации по времени (`TIMEZONE` из `.env`).
- Админ-панель со статистикой, отчётами, статусом прокси, бэкапом базы и последними ошибками публикации.

## Настройка `.env`

```bash
cp .env.example .env
```

Поля:
- `BOT_TOKEN`
- `OWNER_IDS`
- `PROXY_URL` (например `socks5://127.0.0.1:1080`)
- `TIMEZONE`
- `DATABASE_PATH`
- `LOG_LEVEL`

`.env` не должен попадать в git. Реальные токены, ID каналов и секреты не храните в репозитории.

## PROXY_URL и sing-box

Бот использует общий прокси через `AiohttpSession(proxy=PROXY_URL)` (aiogram 3.x).

Важно: второй sing-box не нужен, используется уже существующий общий SOCKS5 прокси. Логика прокси не меняется.

## Как пользоваться

1. Нажмите `➕ Добавить канал`, добавьте бота админом в канал.
2. Нажмите `✍️ Создать пост`.
3. Отправьте:
   - текст (текстовый пост),
   - фото без подписи (фото-пост),
   - фото с подписью (фото+текст).
4. Выберите канал.
5. Проверьте реальный предпросмотр в личке.
6. Выберите действие в inline-меню.

### URL-кнопки

В меню поста нажмите `🔗 URL-кнопки` и отправьте кнопки строками:

```text
Донат | https://example.com/donate
VK | https://example.org
```

Правила:
- максимум 8 кнопок;
- каждая строка должна содержать `|`;
- URL должен начинаться с `http://` или `https://`;
- текст кнопки не должен быть пустым.

### Автоподпись канала

В `📋 Мои каналы` рядом с каналом нажмите `✍️ Подпись`.

- Отправьте текст подписи, чтобы сохранить её для канала.
- Отправьте `-` или `off`, чтобы очистить подпись.
- При публикации подпись добавляется так:

```text
основной текст

подпись
```

### Замена фото

В меню поста нажмите `🖼 Заменить фото` и отправьте новое фото.

- Для photo draft обновляется `media_file_id`, caption остаётся прежним.
- Для text-only draft пост превращается в photo-пост, если текущий текст помещается в лимит caption 1024 символа.

### Дублирование и перепубликация

- `📄 Дублировать` создаёт новый черновик с тем же текстом, медиа, URL-кнопками и выбранным каналом.
- `🔁 Перепубликовать` доступно для опубликованного поста: создаётся новый черновик, затем нужно выбрать активный канал и подтвердить публикацию. Исходный опубликованный пост не меняется.

### Поиск

Нажмите `🔎 Поиск постов`, отправьте запрос — бот покажет последние 10 совпадений только среди ваших постов и добавит кнопки `Открыть #id`.

### Черновики

- Раздел `📝 Черновики` показывает 10 последних черновиков.
- Можно открыть, удалить, отредактировать текст/caption, добавить URL-кнопки, заменить фото, запланировать или опубликовать.
- Кнопка `🧹 Очистить старые черновики` удаляет только ваши draft-посты старше 30 дней после подтверждения.

### Расписание

- Нажмите `🕒 Запланировать`.
- Отправьте дату в формате `ДД.ММ.ГГГГ ЧЧ:ММ` (пример: `19.05.2026 14:30`).
- Фоновый scheduler проверяет очередь каждые 30 секунд и использует тот же механизм публикации, что ручная публикация.

### Лимиты

- Подпись к фото вместе с автоподписью канала не должна превышать 1024 символа.
- Если caption + подпись длиннее лимита, публикация не выполняется и пользователь получает понятное сообщение.

## Админ-панель

OWNER_IDS видят `🛠 Админ-панель`.

- `📊 Статистика`, `📈 Отчёт за 24ч`, `📈 Отчёт за 7д` — агрегированная статистика.
- `👥 Пользователи`, `📣 Каналы` — последние пользователи и каналы.
- `💾 Бэкап базы` — создаёт `backups/bot_backup_YYYY-MM-DD_HH-MM-SS.db` из текущей SQLite-базы и пытается отправить файл администратору документом.
- `🚨 Ошибки` — показывает последние 10 записей `publish_logs` со статусом `error`.
- `🌐 Прокси-статус` — показывает, задан ли `PROXY_URL`.

Бэкапы `backups/` и `backups/*.db` игнорируются git.

## Права в канале

- Бот: `administrator`/`creator`, с правом `can_post_messages`.
- Пользователь: `administrator`/`creator`.

Если публикация не проходит из-за прав, пользователь получает короткое понятное сообщение. Полные технические ошибки пишутся в `logs/bot.log` и `publish_logs`.

## Проверка логов systemd

```bash
sudo systemctl status raccoon-channel-poster
sudo journalctl -u raccoon-channel-poster -n 200 --no-pager
sudo journalctl -u raccoon-channel-poster -f
```
